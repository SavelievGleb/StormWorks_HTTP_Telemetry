# 🚀 StormWorks Telemetry Collector

Гибкая и мощная система для сбора числовых значений в StormWorks и сохранения их на ПК в удобном формате.

Создана, чтобы облегчить жизнь креаторам, предоставляя простой и надёжный доступ к данным с ваших творений для последующего анализа в ваших любимых инструментах.

## 💡 Как это работает?

**Lua** в StormWorks позволяет отправлять *http get запрос на localhost*, которые программа примет, распарсит и сохранит в текстовый файл.

Очень гибкая настройка частоты сбора данных позволяет настроить контроллер под конкретную задачу. В редких случаях возможна сниженная скорость отправки данных, но это решает буфер данных в микроконтроллере.

| Частота сбора | Информативность | Рекомендации |
|:---:|:---|:---|
| **60 Гц** | 🟢 Максимальная информативность. Каждый тик сохраняется | Для данных которые меняются очень резко. Идеально для настройки ракет/фильтров радара |
| **30 Гц** | 🟡 Приличная информативность. Сохраняется каждый второй тик данных | Подойдёт для настроки инертных систем |
| **20 Гц** | 🟠 Информативность ниже среднего. Сохраняется каждый третий тик данных | Для сильно инертных данных, таких как расход топлива или график нагрева двигателя |
| **< 20 Гц** | 🔴 Плохая информативность. | Сам не знаю зачем, пускай будет |
| **0.5 - 0.1 Гц** | 🟤 Всратая информативность. Тут вообще забей | Специально создано для логирования фоновых данных |

**Рекомендую подбирать частоту исходя из задачи и производительности в Вашем конкретном случае*

## 🔧 Установка

1. Необходимо установить [Node JS](https://nodejs.org/) с официального сайта. *Chocolatey не нужен.*
1. Скачать файлы проекта [Releases](https://github.com/SavelievGleb/StormWorks_HTTP_Telemetry/releases)
1. Распаковать файлы проекта в удобную для Вас папку
1. Открываем папку *Server*, запускаем install.bat. По завершении установки (сообщение **Install complete!**) закрываем консоль. Принимающий сервер готов к работе
1. Далее необходимо скопировать контроллер в файлы игры. Открываем папку *microprocessors*, запускаем **copy to game.bat** или можем скопировать *xml* и *png* файлы вручную.

## ▶️ Как пользоваться

1. В папке проекта запускаем **Start server.bat**. Появится консольное окно с надписью *Server started on 127.0.0.1:8080* - сервер работает.
1. В игре:
    1. Поставьте контроллер на постройку
    1. Подключите композитный сигнал к *Data*, кнопку к *Send*
    1. Откройте настройки контроллера и пропишите названия у нужных каналов для сбора. Каналы с пустым именем будут игнорироваться!

Подаём на *Send* сигнал **true** (например включенная кнопка) - начинается сбор и отправка данных. В консоли сервера будет отображаться скорость передачи данных *(frequency)* и кол-во принятых кадров телеметрии. Кадры записываются в *txt* файл, каждый кадр - новая строка. Записанные *txt* файлы храняться в *Server/data*, с форматом имени "*день-месяц-год часы-минуты-секунды (порт сервера)*"
